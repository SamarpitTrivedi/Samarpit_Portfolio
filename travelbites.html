Zomato Data Analysis

CONCEPT

This project uses Power BI to create an interactive sales analysis system for Zomato's restaurant, cuisine, and rating data. The goal is to help stakeholders understand key sales trends and make informed decisions. The dataset, sourced from a publicly available Zomato database, includes attributes like restaurant_name, city, cuisine, and aggregate_rating. Data was prepared in Power Query and modeled using a star schema to integrate various tables for in-depth analysis.

APPROACH

Our approach involved moving from basic visualizations, like identifying top countries by restaurant count, to more analytical insights. We focused on analyzing restaurant and cuisine patterns by segmenting data based on restaurant offerings and customer ratings. The overall process combined data cleaning and feature engineering with DAX-driven measures to create a real-time, interactive environment for monitoring sales trends.

ALGORITHMS & TECHNIQUES USED

To analyze the data, we used:

  • Data Preparation & Modeling: Power Query for data cleaning and transformation, and a star schema for optimized relationships.
  • DAX Calculations: KPI metrics for total restaurants, cities, and cuisines, as well as measures for average cost, cuisine count, and aggregate ratings.
